Docker 簡介

Docker 是一個用於 建構(build) 運行(run) 傳送(share) 應用程序的平台

我們來想像一個情境，在一個前後端分離的專案中，以前端(React)為例，需要安裝 NodeJS、npm 套件，再加上後端自己的啟動環境，甚至有可能還有資料庫，光是要設定這些環境頭都痛了，這時候你就需要 Docker 了

有了 Docker 就可以把在執行程式需要的環境、第三方套件、啟動命令等打包在一起，以便在任何環境都能正確的運行

Docker 概念

先來介紹 Docker 的核心概念

  • images（映像檔）：映像檔是一個只讀的模板，他可以創建容器
  • containers（容器）：容器提供一個獨立可移植的環境，可以在這個環境執行程式

以生活中的例子來看，images 像是像是一個食譜，而 containers 就是根據這個食譜做出來的菜，可以根據食譜做出很多道菜，只要都有嚴格按照食譜做，那麼做出來的菜都會是一樣的。
如果想把這個食譜分享給別人，就可以把它分享到 registry 上，其他人也可以一起使用。最常見的 registry 是 DockerHub，可以在上面分享或下載 images
