CSS 動畫之 Animation

Intro

介紹完了 transition，如果有些動畫還是做不出來，那麼用 animation 就對了

那就開始吧 🐾

@keyframes 關鍵影格

透過許多關鍵影格的組合，動畫的漸變過程

在 CSS 動畫裡，from 表示起始，也可以用 0% 表示，to 表示結束，也可以用 100% 表示
數值範圍是 0%~100%

不可單獨使用數字，一定要加入 %

Animation 屬性

  • animation-name @keyframe 動畫名稱(可以自行取名)
  • animation-duration 動畫播放一次需要的時間（s 或 ms）（預設 0）
  • animation-delay 延後多久（s 或 ms）播放動畫（預設 0）
  • animation-iteration-count 動畫播放次數（預設 1）
  • animation-direction 動畫播放方向（預設 normal）
  • animation-timing-function 動畫播放的速度曲線（預設 ease）
  • animation-fill-mode 動畫播放前後狀態（預設 none）

Animation 語法＆示範

縮寫

這邊比較三種不同 duration 時間和 timing-function(在上一篇有介紹過)

再來講一下在 transition 中沒有的屬性 animation-direction

  • normal：正常播放，從 0% 到 100% ( 預設值 )。
  • reverse：反轉播放，從 100% 到 0%。
  • alternate：正反轉輪流播放，奇數次為 0% 到 100%，偶數次為 100% 到 0%，若動畫播放次數只有一次就只會正常播放。
  • alternate-reverse：alternate 的相反，奇數次為 100% 到 0%，偶數次為 0% 到 100%，若動畫播放次數只有一次就只會反轉播放。

我們來比較一下 normal 和 alternate

有看出差異嗎～ alternate 有點類似來回動作，normal 就是從頭開始在播放

最後來看 animation-fill-mode

  • none：預設值，不論動畫播放次數，結束後一律返回原始狀態。
  • forwards：動畫結束後，保持在最後一個影格狀態。
  • backwards：動畫結束後，保持在第一個影格狀態 ( 但實際測試和 none 效果一樣 )。
  • both：依據動畫的次數或播放方向，保持在第一個影格或最後一個影格狀態，相當實用。

我們比較預設值和 both 的差異（因為是 gif 所以可能有點難看出差異？）

在預設值中，動畫結束後會返回一開始的狀況（以上面例子來看是紅色）
在 both 下，動畫結束後會停在最後一個影格（以上面例子來說，動畫來回三次，最後一格是停在紫色）
